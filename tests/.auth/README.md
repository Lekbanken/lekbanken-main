# Auth State Files

This directory stores Playwright authentication state for different user roles.

**Files created by auth setup:**
- `user.json` - Default user auth (from auth.setup.ts)
- `system-admin.json` - System admin auth (from auth.setup.system-admin.ts)
- `tenant-admin.json` - Tenant admin auth (from auth.setup.tenant-admin.ts)
- `regular-user.json` - Regular user auth (from auth.setup.regular-user.ts)

**Environment Variables Required:**

```bash
# Default user
TEST_USER_EMAIL=test@example.com
TEST_USER_PASSWORD=testpassword123

# System admin (has system_admin role)
TEST_SYSTEM_ADMIN_EMAIL=admin@example.com
TEST_SYSTEM_ADMIN_PASSWORD=adminpassword123

# Tenant admin (has tenant owner/admin role but NOT system_admin)
TEST_TENANT_ADMIN_EMAIL=tenant-admin@example.com
TEST_TENANT_ADMIN_PASSWORD=tenantpassword123

# Regular user (no admin roles)
TEST_REGULAR_USER_EMAIL=user@example.com
TEST_REGULAR_USER_PASSWORD=userpassword123

# Test tenant IDs
TEST_TENANT_ID=your-test-tenant-uuid
OTHER_TENANT_ID=another-tenant-uuid
```

These files are auto-generated by Playwright's auth setup and should be gitignored.
